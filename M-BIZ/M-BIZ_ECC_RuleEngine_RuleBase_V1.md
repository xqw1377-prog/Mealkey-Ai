## 1. 文档概述

### 1.1 文档目的

本文档定义 MealKey ECC 规则引擎的实现规范与规则库定义，是 Founder Intelligence Layer 认知链的工程落地层。它将 FIL 文档中定义的 L2 规则认知、L4 策略匹配、L5 验证动作模板转化为可编码的规则 YAML 定义与引擎 API。

### 1.2 适用范围

| 角色 | 关注重点 |
|------|----------|
| 后端工程师 | 规则引擎实现、API 开发 |
| 知识工程师 | 规则编写、测试与维护 |
| 算法工程师 | 推理逻辑、置信度计算 |
| QA 工程师 | 规则测试用例、回归测试 |

### 1.3 依赖关系

| 文档 | 提供能力 |
|------|----------|
| M-BIZ 专业判断模型设计文档 V1 | 推理引擎架构、证据聚合算法 |
| M-BIZ 商业模式专业判断模型 V1 | 九维评估规则、画像匹配逻辑 |
| MealKey FIL 创业认知链 | 五层认知链定义、验证动作机制 |

### 1.4 术语定义

| 术语 | 定义 |
|------|------|
| 规则(Rule) | 条件-结论对，规则引擎的最小执行单元 |
| 规则集(RuleSet) | 按领域/阶段分组的规则集合 |
| 规则引擎 | 加载、匹配、执行规则的核心运行时 |
| 事实(Fact) | 输入到引擎的数据单元 |
| 推论(Inference) | 规则执行后输出的结论 |
| 验证模板 | 预定义的验证动作生成模板 |
| 追问模板 | 信息不足时触发的问题模板 |

## 2. 规则引擎架构

### 2.1 架构总览

规则引擎位于 MealKey 后端服务中。

```
MealKey API Gateway
    |
    v
+----------------------------+
|   ECC Controller          |
|   层间路由 + 状态管理     |
+----------------------------+
    |           |
    v           v
+----------+ +----------+
|  L1+L2   | |  L3+L4   |
| 规则引擎  | | 分析引擎  |
+----------+ +----------+
    |           |
    v           v
+----------------------------+
|   L5 验证任务管理器        |
|   合规检查 + 回注处理      |
+----------------------------+
    |
    v
+----------------------------+
|   Knowledge Store (KAS)    |
|   规则库 / 画像 / 基准     |
+----------------------------+
```

### 2.2 核心组件

| 组件 | 职责 | 技术选型 |
|------|------|----------|
| RuleLoader | 从 KAS 加载规则集到内存 | YAML + 缓存 |
| FactCollector | 收集并标准化输入事实 | Pydantic 校验 |
| RuleMatcher | 匹配事实与规则条件 | 前向链匹配 |
| InferenceEngine | 执行规则链推理 | 自定义引擎 |
| ConfidenceCalculator | 计算结论置信度 | 加权融合 |
| VerificationGenerator | 生成验证动作 | 模板注入 |
| ComplianceChecker | 检查不算死账合规 | 规则检查器 |

### 2.3 推理流程

```
输入事实
    |
    v
[FactCollector] 校验并标准化
    |
    v
[RuleLoader] 加载匹配的规则集
    |
    v
[RuleMatcher] 遍历规则，匹配条件
    |   |
    |   +-- 条件满足 -- 执行结论
    |   +-- 条件不满足 -- 跳过
    |
    v
[InferenceEngine] 链式推理(最多5层)
    |
    v
[ConfidenceCalculator] 计算置信度
    |
    v
[VerificationGenerator] 生成验证动作(L4)
    |
    v
[ComplianceChecker] 不算死账检查(L5)
    |
    v
输出
```
