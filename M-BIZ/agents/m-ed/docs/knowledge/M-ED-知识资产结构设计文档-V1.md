# M-ED 知识资产结构设计文档 V1

> **版本**：V1.0  
> **更新**：2025-07  
> **状态**：正式发布  

> **说明**：M-ED 为纯规则引擎 + LLM 驱动架构，不依赖外部知识库。知识资产以"规则集"形式存在于代码中。本文档描述这些规则集的结构和内容。

---

## 1. 概述

M-ED 的知识资产目前以 **5 组规则集** 的形式分布在对应的 handler 退路函数中。

| # | 规则集 | 位置 | 规则数 |
|---|--------|------|--------|
| 1 | 股权分配角色权重表 | `handlers/design.py` → `_design_fallback()` | 7 条角色规则 |
| 2 | 动态调整贡献评分规则 | `handlers/adjust.py` → `_adjust_fallback()` | 3 条调整规则 |
| 3 | 场景模拟稀释计算规则 | `handlers/simulate.py` → `_simulate_fallback()` | 2 条计算规则 |
| 4 | 合规检查判定规则 | `handlers/compliance.py` → `_compliance_fallback()` | 5 项 × 3 级判定规则 |
| 5 | 文档生成模板规则 | `handlers/document.py` → `_document_fallback()` | 4 类文档模板 |

## 2. 规则集 1：股权分配角色权重表

### 2.1 角色权重

| 角色 | 权重 | 说明 |
|------|------|------|
| 创始人 | 0.40 | 最高权重，项目发起人和核心决策者 |
| CTO | 0.25 | 技术负责人，技术架构和团队组建 |
| COO | 0.20 | 运营负责人，日常管理和流程建设 |
| 工程师 | 0.15 | 核心技术开发人员 |
| 设计师 | 0.08 | 产品设计和用户体验 |
| 顾问 | 0.05 | 外部顾问，非全职投入 |
| 其他（默认） | 0.10 | 未列出的角色 |

### 2.2 分配公式

```
total_weight = Σ role_weights
创始人的股权占比 = (创始人的权重 / total_weight) × 70%

分配逻辑：
- 团队总分配比例：70%
- 期权池预留：12%
- 未分配：100% - Σ 个人分配 - 12%
- 全职 → 普通股，非全职 → 受限股
```

### 2.3 项目阶段调整

当前规则退路未按阶段差异化分配，建议 LLM 模式下按阶段调整：

| 阶段 | 期权池建议 | 创始人建议最低持股 |
|------|-----------|-------------------|
| idea | 10% | 60% |
| seed | 12% | 50% |
| angel | 15% | 40% |
| pre-a | 18% | 35% |
| a | 20% | 30% |

## 3. 规则集 2：动态调整贡献评分规则

### 3.1 评分公式

```
achievement_weight = min(len(achievements) × 0.5, 5.0)
to_pct = from_pct + achievement_weight
```

| 变量 | 说明 | 约束 |
|------|------|------|
| achievements | 贡献记录中的成就列表 | 每项成就 +0.5% |
| achievement_weight | 总调整权重 | 上限 5.0%，单次调整不超 5 个百分点 |
| from_pct | 当前持股比例 | 从 current_scheme.allocations 中匹配 |
| to_pct | 调整后的持股比例 | from_pct + achievement_weight |

### 3.2 版本规则

```
suggested_version = f"{current_version}.1"
例如：v1 → v1.1, v2 → v2.1
```

### 3.3 调整类型映射

| adjustment_type | 说明 |
|----------------|------|
| general_review | 定期全面回顾，综合多项贡献 |
| milestone | 里程碑达成触发，聚焦特定成就 |
| new_member | 新成员加入，从期权池或未分配中分配 |
| departure | 成员离开，回购或重新分配股权 |

## 4. 规则集 3：场景模拟稀释计算规则

### 4.1 等比稀释公式

```
每轮融资：
  dilution_ratio = dilution_percent / 100
  remaining_ratio = 1 - dilution_ratio
  所有现有股东持股 ×= remaining_ratio
  投资人获得 dilution_percent%
```

### 4.2 约束

- 未分配部分不参与等比稀释（由董事会决定用途）
- 期权池参与等比稀释（视为已预留股份）
- 多轮融资串联执行，每轮独立计算
- 投资人统一命名为 "投资人({round})"

### 4.3 示例

```
初始状态：
  张三: 45%, 李四: 25%, 期权池: 12%, 未分配: 10%

天使轮融资 20%：
  剩余比例 = 80%
  张三: 45 × 0.80 = 36.0%
  李四: 25 × 0.80 = 20.0%
  期权池: 12 × 0.80 = 9.6%
  未分配: 10 × 0.80 = 8.0%
  投资人(天使轮): 20.0%
  合计: 36.0 + 20.0 + 9.6 + 8.0 + 20.0 = 93.6%
  （差额 6.4% 由规则引擎舍入，LLM 模式精度更高）
```

## 5. 规则集 4：合规检查判定规则

### 5.1 Vesting 检查

| 条件 | 结果 | 说明 |
|------|------|------|
| 有 Vesting 条款 + total ≥ 36 且 cliff ≥ 6 | pass | 条款合理 |
| 有 Vesting 条款 + 不满足上述条件 | warn | 周期偏短，建议至少 48/12 |
| 无 Vesting 条款 | fail | 缺少创始人保护核心机制 |

### 5.2 股权类型检查

| 条件 | 结果 | 说明 |
|------|------|------|
| 分配中包含 ≥ 2 种股权类型 | pass | 类型区分清晰 |
| 所有成员为同一类型 | warn | 建议全职/兼职/顾问区分对待 |

### 5.3 IP 转让检查

| 条件 | 结果 | 说明 |
|------|------|------|
| 含受限股 | warn | 建议在协议中明确知识产权转让条款 |
| 不含受限股 | warn | 建议在股东协议中明确知识产权转让条款 |

### 5.4 税务检查

| 条件 | 结果 | 说明 |
|------|------|------|
| 始终 | warn | 建议咨询税务专业人士 |

### 5.5 创始人保护检查

| 条件 | 结果 | 说明 |
|------|------|------|
| 最高持股 ≥ 51% | pass | 拥有绝对控制权 |
| 最高持股 ≥ 34% 且 < 51% | warn | 无绝对控制权，建议 AB 股或一致行动协议 |
| 最高持股 < 34% | fail | 创始人控制权严重不足 |
| 无分配数据 | warn | 无法评估 |

### 5.6 综合判定

| 条件 | overall_status |
|------|---------------|
| 存在 fail 项 | fail |
| 无 fail，存在 warn 项 | caution |
| 全部 pass | pass |

## 6. 规则集 5：文档生成模板规则

### 6.1 文档类型

| document_type | 文档名称 |
|--------------|----------|
| equity_agreement_draft | 股权分配协议（草案） |
| board_resolution | 董事会决议 |
| shareholder_agreement | 股东协议 |
| summary_report | 总结报告 |

### 6.2 模板结构

规则退路仅生成框架（草案级别），包含：
1. **协议主体** — 签约各方基本信息
2. **股权分配** — 分配明细（占位符）
3. **免责声明** — 不构成法律意见

LLM 模式生成完整文档，包含详细条款。

### 6.3 通用模板

```markdown
# {project_name} 股权分配协议（草案）

## 一、协议主体

本协议由以下各方于 {date} 共同签署：

- {name}（{role}，持股 {equity_percent}%）

## 二、股权分配

...（详细条款待 LLM 生成）...

---

> **免责声明**：本文件为 AI 生成的参考草案，不构成法律意见。
```

- **中文关键词**：字符子串匹配（中文以词为单位）
