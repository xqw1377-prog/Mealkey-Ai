## 概述

本版本定义 **专业判断结构**，不限定具体数据源。后续可以把城市画像、平台数据与案例库逐步接入，但不改变六维模型与机会评分的表达方式。

## 六维模型定义

六维模型是专业判断结构的核心，包含六个标准化的评估维度。

### 维度定义模板

每个维度包含：
- **维度名称**
- **维度描述**
- **评估指标列表**（每个维度 5 个指标）
- **权重 (0-1)**（六个维度权重之和为 1.0）
- **评分标准**

### 六维列表

| # | 维度名称 | 权重 | 核心评估问题 |
|---|----------|------|-------------|
| 1 | **市场容量** | 0.25 | 这个市场有多大？还能长多大？ |
| 2 | **竞争格局** | 0.20 | 这个市场竞争有多激烈？还有没有位置？ |
| 3 | **消费适配** | 0.20 | 这里的消费者吃这一套吗？ |
| 4 | **运营可行性** | 0.15 | 这个生意做得起来吗？供应链撑得住吗？ |
| 5 | **品牌势能** | 0.10 | 这个品牌能打赢吗？有什么核心优势？ |
| 6 | **环境适配** | 0.10 | 大环境支持吗？政策、趋势、风险如何？ |

六个维度通过 `DimensionRegistry` 统一注册，权重由餐饮行业经验判断决定。

## 机会评分模型

机会评分基于六维模型的加权综合评分。

### 评分公式

```
机会评分 = Σ(维度_i得分 × 维度_i权重) / Σ(维度_i权重)
维度得分  = Σ(指标_i归一化得分 × 指标_i权重) / Σ(指标_i权重)
```

### 评分范围

- **0.0 - 2.0**: 低机会
- **2.0 - 3.5**: 中机会
- **3.5 - 5.0**: 高机会

### 维度得分等级

| 得分 | 等级 | 说明 |
|------|------|------|
| 4.0 - 5.0 | 优秀 | 显著优势 |
| 3.0 - 4.0 | 良好 | 具备优势 |
| 2.0 - 3.0 | 一般 | 行业平均 |
| 1.0 - 2.0 | 较弱 | 存在短板 |
| 0.0 - 1.0 | 差 | 明显劣势 |

## 知识资产（已内置）

本版本内置以下知识资产，不依赖外部数据源：

1. **品类知识卡** × 10（湘菜、火锅、烧烤、咖啡、茶饮、快餐、烘焙、面馆、日料、西餐）
2. **城市画像** × 10（长沙、成都、上海、深圳、杭州、广州 等）
3. **案例研究** × 8（海底捞、瑞幸、巴奴、蜜雪冰城、社区餐饮、老乡鸡、太二酸菜鱼、费大厨、紫光园）
4. **判断规则** × 10（品类规则、创业者规则、城市规则、组合规则、案例规则）

## 数据源接入策略

后续接入包括：

1. **城市画像** — 城市级别的宏观数据（已预留 `CityProfileAdapter`）
2. **平台数据** — 业务平台运营数据（已预留 `PlatformDataAdapter`）
3. **案例库** — 历史案例与标杆对比（已预留 `CaseLibraryAdapter`）

所有数据源通过 **适配器层** 接入，不侵入核心模型。

## 使用方式

### Python API

```python
from core import MKTEngine

engine = MKTEngine()
decision = engine.analyze("咖啡", "上海", experience="丰富", capital_level="high")

print(decision.opportunity_score)  # 机会评分
print(decision.opportunity_level)  # 机会等级
print(decision.positioning_suggestions)  # 策略建议
```

### 命令行

```bash
python -m core.cli 咖啡 上海
python -m core.cli 火锅 成都 --experience 首次创业 --capital low
python -m core.cli --list-categories
python -m core.cli 湘菜 --json
```
