## 0. 核心原则

> 用户不是来「看数据」的，而是来「推进企业决策」的。

```text
企业现实
  → 问题发现
  → 专家判断
  → 决策形成
  → 执行验证
  → AI 成长
```

从 **AI Dashboard** → **AI 咨询工作空间**。

不是极简空壳 SaaS。  
不是 ChatGPT。  
是 Founder Intelligence Layer 的前台。

## 2. 五页骨架

### 2.1 今日判断（Home）

**不是**首页 Dashboard。  
**是**顾问简报。每页只有一个主 CTA。

```text
早上好，王总

我重新审视了你的企业。

今天最值得讨论的问题：
「加盟扩张是否已经成熟？」

风险：组织复制能力不足
建议：召开商业战略会议

[进入会议]

────────
最近变化：
✓ 完成第二家店验证
↑ 品牌搜索增长
↓ 人效下降

────────
正在推进：品牌定位项目
阶段：市场研究完成
下一步：专家定位会议
```

禁止：10 个入口、健康度大数字、驾驶舱文案、专项入口抢首页。

---

### 2.2 企业世界（World）

**不是**企业管理后台。  
**是** AI 的企业模型。

**列表：**

```text
我的企业

湘宴 · 餐饮 · 扩张期
AI判断：单店模型成立，复制能力不足
当前议题：是否进入加盟
[进入世界 >]
```

**详情：**

```text
湘宴 · 企业状态：扩张验证期

AI理解：
你是谁：高端湘菜品牌
优势：产品力 / 供应链
风险：复制体系

当前核心问题：
如何从一家店变成 100 家店

相关咨询（等权，此处才出现四部门）：
商业模式 | 品牌定位 | 组织设计 | 市场研究
```

注意：M-BIZ / M-PNT / M-ED / M-MKT **出现在企业世界**，不出现在今日首页。

创建企业后：先进企业世界（访谈/成型），不直跳会议。

---

### 2.3 咨询会议（核心）

**会议最重要，不是首页最重要。**

默认状态 = 会议室，不是聊天。

```text
商业战略会议

议题：是否启动加盟扩张？

── 专家席 ──
商业模式顾问 · 暂缓加盟
原因：1) 单店盈利未稳 2) 组织能力不足

增长顾问 · 市场窗口存在
运营顾问 · 需要复制体系

── 会议结论 ──
90 天验证直营复制能力

[接受方案]  [继续讨论]

补充意见：（输入框，次要）
```

AI 不是一个人，而是一组专家。  
输入框是补充，不是主体。

---

### 2.4 决策档案

**不是**任务列表 / 统计墙。  
**是**企业历史选择记录。

```text
D-001 是否加盟扩张 · 暂缓 · 验证中
验证：3 个月完成 3 店复制

D-002 品牌重新定位 · 完成
```

详情必须能回答：

- 为什么这么决定？
- 有哪些争议？
- 放弃了什么？
- 如何验证？
- 结果如何？

→ Decision Memory。

---

### 2.5 AI认知

禁止：能力指数、100 分、产品内部链路。

```text
我对你的理解

经营风格：稳健增长型
优势：运营经验丰富
盲区：品牌战略投入不足

企业变化：
6 个月前：单店经营
现在：进入复制阶段

我建议下一步：建立品牌增长体系
```

这是用户感知「AI 越来越懂我」的地方。

## 3. 四个 Agent = 四个咨询部门

不是四个独立产品。是 Founder OS 下的四个部门。

```text
企业世界 → 咨询会议 → 选择部门（或从世界等权进入部门会议室）
```

| Agent | 部门 | 输出 | 会议形式 |
|-------|------|------|----------|
| M-MKT | 市场研究部 | 市场机会判断 | 市场分析会 |
| M-PNT | 品牌战略部 | 品牌定位 | 定位委员会 |
| M-BIZ | 商业战略部 | 商业模式优化 | 商业模型评审 |
| M-ED | 组织设计部 | 股权/组织方案 | 组织设计会议 |

可独立跑、可挂母体；共享企业上下文；**不做**强制串联流水线。

各部门共用同一套会议母版（议题 / 专家席 / 结论 / 补充输入），只换部门议题域。

## 4. 视觉语言（方向冻结，细节后做）

统一 Founder OS，禁止各部门各一套主色：

```text
米白背景
深灰文字
黑色行动按钮
少量品牌色仅作部门标识
```

气质：咨询公司工作空间，不是 SaaS 控制台。

---

## 5. 交互规则冻结

1. **每页一个核心问题** — 不展示 20 个能力  
2. **每一步必须用户参与** — AI 给方向，用户选择继续哪个  
3. **任何输出必须进入下一步** — 结束不是下载 PDF，而是进入下一次会议/决策  

---

## 6. 用户旅程（产品形态）

**第一次：**

```text
创建企业 → AI访谈 → 形成企业世界
  → 发现第一个问题 → 第一次咨询会议 → 第一个决策
```

**长期：**

```text
AI 越来越懂企业 → 越来越懂老板 → 越来越能提前发现问题
```
