# M-PNT V2 最终收口定义

> 模块：M-PNT
> 英文全称：Market Positioning Think Tank
> 中文定位：AI 原生品牌战略咨询工作台
> 文档角色：V2 最终收口裁决稿

## 一、产品重新定义

### 原来的理解

```text
用户输入品牌信息
  ↓
AI 分析
  ↓
生成定位方案
```

问题：

- 用户参与感低
- AI 像报告机器
- 专家像三个 Prompt
- 结果不可沉淀
- 没有战略过程

### 新定义

```text
创始人进入品牌战略咨询流程
  ↓
AI 理解企业
  ↓
市场洞察
  ↓
理论专家分析
  ↓
专家冲突
  ↓
战略共识
  ↓
定位冻结
  ↓
执行路径
```

M-PNT 不输出一个答案。

它帮助用户完成一次：

**品牌战略决策。**

## 二、M-PNT 核心链路冻结

最终流程：

```text
01 品牌理解
      |
      |
02 市场洞察
      |
      |
03 定位专家委员会
      |
      |
04 AI 战略会议
      |
      |
05 定位契约
      |
      |
06 执行路径
```

对应用户心理：

| 阶段 | 用户感受 |
| --- | --- |
| 品牌理解 | AI 开始懂我 |
| 市场洞察 | AI 看到了我没看到的问题 |
| 专家分析 | 不同高手帮我思考 |
| 战略会议 | 我的方案被挑战和优化 |
| 定位契约 | 我形成了战略共识 |
| 执行路径 | 我知道下一步怎么做 |

## 三、三个定位专家重新定义

冻结：

专家不是：

- 三个内容生成器

而是：

- 三种战略认知模型

### Ries

代表：

## 心智占位模型

关注：

- 消费者脑中有没有位置
- 能否形成简单认知
- 是否具备第一属性

问题：

> 用户为什么记住你？

### Trout

代表：

## 竞争定位模型

关注：

- 竞争差异
- 对手空位
- 品类关系

问题：

> 为什么选择你而不是别人？

### 叶茂中

代表：

## 消费者洞察模型

关注：

- 情绪需求
- 场景
- 文化表达

问题：

> 为什么消费者愿意喜欢你？

最终：

三个专家不是争论谁正确。

而是：

**形成战略平衡。**

## 四、AI Chief Advisor 重新定位

这是最关键的一点。

Chief 不应该：

- 替用户回答
- 直接生成方案

而应该是：

## 战略主持人

负责：

### 1. 收集信息

知道老板是谁。

知道企业阶段。

知道资源。

知道目标。

### 2. 发现缺口

例如：

> 你说想做年轻湘菜，但目前没有明确年轻消费者画像。

### 3. 组织专家

让不同理论碰撞。

### 4. 推动决策

最终：

> 我建议冻结这个方向，原因如下。

## 五、后端能力重新映射

当前后端：

```text
Capability

Theory Matrix

Decision Engine

Cross Fire

Knowledge

Runtime
```

全部保留。

但是产品映射改变。

以前：

```text
Capability
  ↓
输出
```

现在：

```text
战略阶段

↓

调用 Capability

↓

形成 Decision Artifact
```

例如：

市场分析能力以前输出市场报告。

现在成为：

> 市场洞察阶段资产

专家矩阵以前生成专家观点。

现在成为：

> 战略会议输入

Decision Engine 以前做评分判断。

现在成为：

> 战略主持大脑

## 七、用户数据飞轮

这是 AI 越来越懂用户的关键。

以后每一次使用，都应沉淀：

## Founder Memory

包括：

### 创业者认知

- 喜欢什么打法
- 风险偏好
- 资源优势

### 品牌认知

- 品牌阶段
- 历史决策
- 曾经尝试

### 决策模式

- 接受什么建议
- 拒绝什么建议
- 为什么

最终：

AI 不只是懂餐饮。

而是：

**懂这个老板。**

## 八、V2 后 M-PNT 完整定位

最终一句话：

> M-PNT 是一个 AI 品牌战略咨询机构，它通过市场分析、定位理论专家协作和战略推演，帮助创业者完成从品牌认知到定位决策再到执行落地的完整过程。
