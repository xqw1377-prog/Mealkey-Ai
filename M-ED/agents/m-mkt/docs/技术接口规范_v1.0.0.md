## 1. 概述

本文档定义 M-MKT 餐饮市场机会分析引擎的技术接口规范。

核心定位：
- **专业判断结构** — 基于领域专业知识定义评估框架
- **数据源无关** — 不限定具体数据源，后续可接入城市画像、平台数据、案例库
- **表达稳定** — 六维模型与机会评分的表达方式不随数据源变化

## 2. 核心数据模型

### 2.1 SixDimensionModel（六维模型）

六维模型是整个系统的核心数据结构，包含六个标准化的评估维度。

```
SixDimensionModel
├── name: str                    # 模型名称
├── description: str             # 模型描述
└── dimensions: List[Dimension]  # 六个维度（按 id 排序）
```

### 2.2 Dimension（维度）

```
Dimension
├── id: int           # 维度编号 (1-6)
├── name: str         # 维度名称
├── description: str  # 维度描述
├── weight: float     # 维度权重 (0-1)
├── score: float?     # 维度得分 (0-5)，由 calculate_score() 计算
└── indicators: List[Indicator]  # 维度内指标列表
```

### 2.3 Indicator（指标）

```
Indicator
├── name: str               # 指标名称
├── description: str        # 指标说明
├── value: float?           # 原始值
├── normalized_score: float? # 归一化得分 (0-5)
└── weight: float           # 在维度内的权重 (0-1)
```

### 2.4 ScoreLevel（评分等级枚举）

| 枚举值 | 得分范围 | 说明 |
|--------|----------|------|
| EXCELLENT | 4.0 - 5.0 | 优秀 |
| GOOD | 3.0 - 4.0 | 良好 |
| AVERAGE | 2.0 - 3.0 | 一般 |
| WEAK | 1.0 - 2.0 | 较弱 |
| POOR | 0.0 - 1.0 | 差 |

## 3. 评分引擎 API

### 3.1 ScoreEngine 类

评分引擎负责所有评分计算逻辑，不依赖任何外部数据源。

#### 3.1.1 normalize_score

```
normalize_score(value: float, min_val: float, max_val: float) -> float
```

线性归一化，将原始值映射到 0-5 分。

- 输入：原始值、最小值基准、最大值基准
- 输出：0.0 - 5.0 的归一化得分
- 边界：value ≤ min_val → 0.0；value ≥ max_val → 5.0

#### 3.1.2 normalize_score_reverse

```
normalize_score_reverse(value: float, min_val: float, max_val: float) -> float
```

反向线性归一化（值越小得分越高），将原始值映射到 0-5 分。

- 输入：原始值、最小值基准、最大值基准
- 输出：0.0 - 5.0 的归一化得分
- 边界：value ≤ min_val → 5.0；value ≥ max_val → 0.0

#### 3.1.3 evaluate_model

```
evaluate_model(model: SixDimensionModel) -> dict
```

评估六维模型，返回完整评分报告。

**返回格式：**
```json
{
  "model_name": "六维模型",
  "opportunity_score": 3.25,
  "opportunity_level": "中机会",
  "dimensions": [
    {
      "id": 1,
      "name": "维度一",
      "score": 3.5,
      "level": "良好",
      "weight": 0.2,
      "indicators": [
        {
          "name": "指标A",
          "normalized_score": 3.5,
          "weight": 1.0,
          "weighted_score": 3.5
        }
      ]
    }
  ]
}
```

## 4. 数据源适配器接口

### 4.1 BaseAdapter（抽象基类）

所有数据源适配器必须继承 BaseAdapter 并实现以下接口。

```python
class BaseAdapter:
    config: AdapterConfig
    
    # 必须实现的抽象方法
    def fetch(self, **kwargs) -> Any          # 从数据源获取原始数据
    def transform(self, raw_data) -> Dict     # 转换为标准化指标数据
    
    # 可选覆盖
    def validate(self) -> bool                # 验证配置有效性
    def get_info(self) -> Dict                # 获取适配器信息
```

### 4.2 AdapterConfig

```python
AdapterConfig:
    name: str          # 适配器名称
    version: str       # 版本号
    enabled: bool      # 是否启用
    params: Dict       # 配置参数
```

### 4.3 预留适配器

| 适配器 | 状态 | 说明 |
|--------|------|------|
| CityProfileAdapter | 预留 | 城市画像数据 |
| PlatformDataAdapter | 预留 | 平台运营数据 |
| CaseLibraryAdapter | 预留 | 案例库数据 |

## 5. 机会评分模型

### 5.1 评分公式

```
机会评分 = Σ(维度_i得分 × 维度_i权重) / Σ(维度_i权重)
```

### 5.2 机会等级判定

| 机会评分范围 | 等级 | 说明 |
|--------------|------|------|
| 3.5 - 5.0 | 高机会 | 市场进入条件优良 |
| 2.0 - 3.5 | 中机会 | 市场具备一定潜力 |
| 0.0 - 2.0 | 低机会 | 市场条件不充分 |

### 5.3 维度得分计算

```
维度得分 = Σ(指标_i归一化得分 × 指标_i权重) / Σ(指标_i权重)
```

## 7. 扩展指南

### 7.1 接入新数据源

1. 创建新的适配器类，继承 `BaseAdapter`
2. 实现 `fetch()` 方法获取原始数据
3. 实现 `transform()` 方法将原始数据转换为 `Indicator` 格式
4. 将指标注入到对应的 `Dimension` 中
5. 调用 `ScoringEngine.evaluate_model()` 获取评估结果

### 7.2 自定义维度

1. 创建 `Dimension` 实例，设置 id、name、description、weight
2. 添加 `Indicator` 列表
3. 调用 `dimension.calculate_score()` 计算维度得分
4. 将维度加入 `SixDimensionModel`
