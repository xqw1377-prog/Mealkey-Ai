## 使用说明

- `[主]` = 唯一主 CTA（黑底）  
- `[次]` = 次要（描边或文字）  
- `···` = 可折叠，默认收起  
- 线框按 **移动优先**；桌面 = 同结构加宽，不增加模块

# 页面 3 · 咨询会议（价值中心）

**路由建议：** `/projects/{id}/advisor`  
**导航标签：** 咨询会议  

## 线框（默认 = 会议室，非 Chat）

```text
┌──────────────────────────────────────┐
│ 退出会议 ←     商业战略会议           │
├──────────────────────────────────────┤
│ 议题                                 │
│ 是否启动加盟扩张？                    │
│                                      │
│ ── 专家席 ─────────────────────────  │
│                                      │
│ ┌ 商业模式顾问 ───────────────────┐  │
│ │ 观点：暂缓加盟                   │  │
│ │ 原因：                           │  │
│ │  1. 单店盈利模型未稳定            │  │
│ │  2. 组织能力不足                 │  │
│ └─────────────────────────────────┘  │
│                                      │
│ ┌ 增长顾问 ───────────────────────┐  │
│ │ 观点：市场窗口存在               │  │
│ └─────────────────────────────────┘  │
│                                      │
│ ┌ 运营顾问 ───────────────────────┐  │
│ │ 观点：需要复制体系               │  │
│ └─────────────────────────────────┘  │
│                                      │
│ ── 会议结论 ───────────────────────  │
│ 90 天验证直营复制能力                 │
│                                      │
│ [主] 接受方案    [次] 继续讨论        │
│                                      │
│ ··· 顾问材料（企业上下文 / 证据）      │
├──────────────────────────────────────┤
│ 补充意见                             │
│ ┌─────────────────────────┐ [发送]   │
│ │ 输入框（次要）            │         │
│ └─────────────────────────┘         │
└──────────────────────────────────────┘
```

## 未形成结论时

- 专家席可先出 1–2 位顾问观点  
- 结论区显示「讨论中 · 待收口」  
- 主 CTA 可为「请顾问收成结论」或「形成经营决策」

## 交互

| 动作 | 结果 |
|------|------|
| 接受方案 | → 写入决策档案，可跳决策详情 |
| 继续讨论 | 焦点到输入框；或抛出下一追问（用户选择方向） |
| 发送补充 | 追加一轮；更新专家席/结论，**不**以气泡流为主舞台 |
| 顾问材料 | 抽屉/底部展开，不挡议题 |

## 部门会议室变体

同一线框，仅改标题与专家构成：

| 部门 | 会议标题示例 |
|------|----------------|
| 市场研究部 | 市场分析会 |
| 品牌战略部 | 定位委员会 |
| 商业战略部 | 商业模型评审 |
| 组织设计部 | 组织设计会议 |

## 禁止

欢迎长气泡、emoji 流、多层折叠藏议题、无后端假反馈。

# 跨页交互总表

| 从 | 到 | 带什么 |
|----|-----|--------|
| 今日判断 → 会议 | 今日核心问题作议题 | |
| 企业世界 → 会议 | 当前核心问题作议题 | |
| 企业世界 → 部门 | 同会议母版 + 部门上下文 | |
| 会议 → 决策 | 接受方案写入 Decision Memory | |
| 决策 → 会议 | 带着该决策继续验证/复盘 | |
| AI认知 → 今日 | 建议下一步映射到简报 | |

# 输出必须进入下一步（规则落地）

| 场景 | 错误结束 | 正确结束 |
|------|----------|----------|
| 市场分析结束 | 下载 PDF | [主] 进入定位委员会 / 带回企业世界 |
| 会议有结论 | 仅「谢谢」 | [主] 接受方案 → 决策档案 |
| 决策验证到期 | 无动作 | [主] 回填结果 / 再开会 |

# 与专项套入关系

五页母版冻结后：

1. 四部门 **不新开产品壳**，只用「咨询会议」母版变体  
2. 部门入口只挂在 **企业世界详情**（及会议材料区）  
3. 访谈/单问单答（如 M-PNT）嵌在会议中间区「当前任务」槽位，不另起 Dashboard
