## 〇、为什么不能跟决策室拼在一起（冻结）

| 表面 | 职责 | 老板心智 |
|------|------|----------|
| **今日雷达** | 发现变化（1 条主焦点） | 「今天有没有事」 |
| **经营分析** | 把这条变化讲清楚（标准报告） | 「到底怎么回事」 |
| **决策室** | 议题确认 → 七常委 → **唯一拍板点** | 「我要不要做」 |

**拍板铁律（冻结）：** 老板签字/做不做/条件做，**只发生在决策室**。今日页、变化解读页、能力页、档案列表均不得出现「拍板 / 签字」主 CTA。

**禁止：**

1. 在决策室写「今日已采齐」——采集/分析不是开案仪式  
2. 「进入经营分析」直跳决策室（跳过中间层）  
3. 用语音开案 Brief 冒充经营分析报告  
4. 分析页堆七常委席位、聊天输入、KPI 大屏  
5. 分析页或今日页伪装「拍板」按钮（只能「进入决策室」）

**正确链路（冻结）：**

```text
今日雷达
  → [进入经营分析]  →  《经营分析》标准页
  → [升格决策室]    →  决策室（议题已就绪）
  → 签字            →  M-EXEC → D+7 复盘
```

旁路：老板也可「自己发起决策」进空白决策室；**不**抢雷达主 CTA。

## 二、标准输出字段（冻结）

字段与 Signal 契约对齐；UI **不得**私增「人格 / Agent 名 / 模型痕迹」。

```typescript
type BusinessAnalysisOutputV1 = {
  // —— 页眉 ——
  brandLine: string;           // 「张小菜 · 经营分析」
  asOfLabel: string;           // 「截至今日观察」
  signalTypeLabel: string;     // 顾客 / 经营 / 竞争 / …
  severityLabel: string;       // 需要决策 | 值得关注 | 正向变化
  importanceStars: 1 | 2 | 3 | 4 | 5;

  // —— 主结论区（首屏） ——
  decisionQuestion: string;    // 老板要回答的一句话问题
  headlineJudgment: string;    // 「我的判断」一句话

  // —— 五层正文（顺序冻结） ——
  observation: string;         // 看到了什么（事实）
  pattern: string;             // 异常/模式
  meaning: string;             // 跟你店的关系
  impact: string;              // 若不管会怎样
  recommendation: string;      // 建议先做什么（非最终拍板）

  // —— 证据 ——
  evidence: Array<{
    claim: string;
    source: string;            // 「公开检索」「周报上传」「Brain」
    kind: "internal_fact" | "external_intel" | "inference";
  }>;                          // ≥2 条方可「需要决策」主推；否则降级观察

  // —— 边界 ——
  known: string[];             // ≤3
  unknown: string[];           // ≤3 · 必须显式
  confidenceNote: string;      // 「证据偏薄 / 可核验」等一句

  // —— 今日动作（可选） ——
  todayOneThing?: { action: string; why: string } | null;

  // —— CTA ——
  primaryCta: "promote_council" | "keep_watching";
};
```

**空态规则：** `evidence.length === 0` 时，正文只保留「暂无可核验线索」+ 补身份/周报引导；**禁止**字源/百科/无关填充。

## 三、视觉结构（冻结线框）

一屏一叙事；**禁止卡片墙**。首屏只留：页眉 · 问题 · 判断 · 五层摘要 · 主 CTA。

```text
┌──────────────────────────────────────────────────────────┐
│  ← 返回今日                        餐启 · 经营分析        │
│  张小菜 · 顾客信号 · ★★★★☆ · 需要决策                      │
│  截至今日观察                                            │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  今天要回答的问题                                        │
│  「最新顾客评价有变化，今天要不要立刻行动？」              │
│                                                          │
│  我的判断                                                │
│  服务体验承压，值得先核对晚市出餐，而不是先改菜牌。        │
│                                                          │
├─ 发生了什么 ─────────────────────────────────────────────┤
│  ① 观察   ……（事实，来自证据）                            │
│  ② 模式   ……                                              │
│  ③ 意义   ……（跟本店关系）                                │
│  ④ 影响   ……                                              │
│  ⑤ 建议   ……（可执行的下一步，非签字）                    │
│                                                          │
├─ 证据（可核验）──────────────────────────────────────────┤
│  · [内部] 周营业额 …                                     │
│  · [外部] 公开检索片段 …                                 │
│  · [推理] 仅作辅助，单独不成链                            │
│                                                          │
├─ 已知 / 未知 ────────────────────────────────────────────┤
│  已知：…    未知：…                                       │
│  置信：证据可核验 / 偏薄，建议先观察                      │
│                                                          │
│  ┌ 今天只做一件事 ────────────────────────────────────┐  │
│  │ 检查 18:00–20:00 出餐节奏                           │  │
│  └────────────────────────────────────────────────────┘  │
│                                                          │
│  [ 升格到决策室 → ]          [ 继续观察 ]                 │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 视觉纪律

| 要 | 不要 |
|----|------|
| 品牌店名在页眉一级 | 只在导航露品牌 |
| 一个主问题压过一切 | 多个并列议题 |
| 五层用数字序号竖排 | 仪表盘 / 四宫格 KPI |
| 证据标明 internal / external / inference | 无来源断言 |
| 主 CTA = 升格决策室（仅 severity=decide） | 「开始决策」伪装成已采齐 |
| 次 CTA = 继续观察 / 返回今日 | 常用题快捷条抢分析页 |

动效（实现时）：进入页眉淡入 → 五层逐行显现 → CTA 最后出现（2–3 段即可）。

## 五、验收（老板手测）

1. 今日点「进入经营分析」→ 落在分析页，**不是**决策室  
2. 分析页可见五层 + 证据来源标签；无证据时诚实空态  
3. 「升格到决策室」后决策室显示「分析已就绪」，可开常委  
4. 「继续观察」回今日，无 DRAFT Decision  
5. 决策室不再出现「今日采集」话术
