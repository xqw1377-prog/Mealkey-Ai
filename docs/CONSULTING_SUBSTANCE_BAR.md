# 咨询价值硬门槛（反玩具感）

> 冻结日：2026-07-15  
> 背景：六步流程已通，但产出若是可替换文案，体验仍是玩具。

## 1. 问题定性

流程正确 ≠ 有咨询价值。

玩具特征：

- 换城市/品类，句子几乎不变
- 三/四顾问「都挺好」，无互斥
- 报告像作文，没有否决条件与本周动作
- 老板看完仍不知道周一干什么、什么情况下该停

## 2. 决策级资产硬门槛（缺一不可）

| # | 资产 | 验收 |
|---|------|------|
| 1 | 本轮唯一问题 | 调研第一屏可见，写清「回答什么」 |
| 2 | 所以呢 | 调研必须有决策含义，不是信息堆 |
| 3 | 顾问互斥 | 冲突摘要含「不能同时为真」；每策 doNotDo 反对邻策 |
| 4 | 取舍 | 决策包写清：得到什么 / 放弃什么 |
| 5 | 否决条件 | ≥2 条可观察停手线 |
| 6 | 本周动作 | ≥3 条周一可执行（门禁：`assertSeatDecisionReady` / 签字 `decision.complete`） |
| 7 | 证据链 | 建议能追溯到调研/证明点 |

## 3. 代码落点

- `packages/agents/src/consulting-os/substance.ts`（生成时补齐 ≥3 本周动作；互斥摘要）
- `packages/agents/src/consulting-os/delivery-gates.ts`（签字检查真实调研状态；顾问互斥；本周≥3）
- `agent-consulting.service` 硬化 research/advisors；确认策略时生成 `decisionArtifact`
- UI：调研突出唯一问题/所以呢；报告改为决策包（否决条件 + 本周动作）
