# Founder OS 五个页面低保真原型 V1

> 配套：`FOUNDER_OS_FRONTEND_MASTER_V2.md`  
> 日期：2026-07-13  
> 目标：**冻结交互，不追求视觉**  
> 回答：用户每天打开系统，到底怎么走？  
> 核心变化：从「浏览 AI 能力」→「推进一次企业决策」  
> 下一层：`FOUNDER_OS_MEETING_SYSTEM_V1.md`（会议系统详细设计）

---

## 原则（三页共通）

1. **第一句话必须有价值** — 禁止「欢迎回来」；必须「我发现了一个问题」  
2. **只给一个主动作** — 禁止四按钮并列  
3. **数字弱化** — 判断才是主角  

---

```text

━━━━━━━━━━━━━━━
```

```text
会议记录

「当前收入模型不足以支撑快速复制。」

━━━━━━━━━━━━━━━
```

主舞台必须含：**核心分歧（Conflict）** + 共识 + 接受/修改/继续。  
详细状态机 / 回合制 / 五节点 / 决策卡 → `FOUNDER_OS_MEETING_SYSTEM_V1.md`。

# 页面 4：决策档案（Decision）

**定位：** 企业的战略历史。未来 AI 最懂你的地方，是决策历史，不是知识库。

```text
决策档案 · 全部决策

━━━━━━━━━━━━
D-001 是否加盟扩张
结论：暂缓 · 原因：模型复制能力不足 · 状态：验证中
查看详情 >

━━━━━━━━━━━━
D-002 品牌重新定位
结论：年轻化升级 · 状态：完成
```

**详情必须回答：**

- 为什么这么决定？  
- 支持 / 反对观点各几条？  
- 最终选择了什么、放弃了什么？  
- 如何验证？结果如何？

# 七、专项 Agent 如何嵌入

用户**看不到**「M-BIZ 产品 / M-PNT 产品」。  
用户看到的是**当前企业问题**。

| 今日判断呈现 | 进入 | 背后 |
|--------------|------|------|
| 品牌认知不足 | 品牌战略会议 | M-PNT |
| 商业模式风险 | 商业战略会议 | M-BIZ |
| 组织扩张风险 | 组织设计会议 | M-ED |
| 市场机会不明 | 市场分析会 | M-MKT |

**Agent 不是产品。Agent 是企业里的专家。**
