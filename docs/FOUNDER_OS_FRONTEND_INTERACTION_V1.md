# Founder OS 前端交互设计 V1

> 日期：2026-07-13  
> 目标：让用户感受到「AI 咨询团队正在工作」  
> 配套：`FOUNDER_OS_MEETING_SYSTEM_V1.md` · `FOUNDER_OS_USER_JOURNEY_V1.md`

---

## 认知转换（冻结）

| 错误感知 | 正确感知 |
|----------|----------|
| 我在问 AI 问题 | 我的企业正在接受一次 AI 顾问诊断 |
| 聊天 → 报告 | 企业 → 问题 → 会议 → 决策 → 成长 |

用户核心动作：**发起一个商业议题**，不是「聊天」。

---

## 主交互模型

```text
企业 → 问题 → 会议 → 决策 → 成长
```

禁止：首页 → Agent 列表 → 聊天 → 报告（工具心智）。

---

## 五页角色

| 页 | 名称 | 用户感觉 |
|----|------|----------|
| 1 | 今日顾问简报 | AI 已经在思考 |
| 2 | 我的企业 | AI 有长期记忆 |
| 3 | 咨询会议 | 顾问团队在开会 |
| 4 | 战略档案 | 我有决策历史 |
| 5 | AI眼中的你 | 它真的了解我 |

---

## Mission 理解流

用户只说一句目标，例如：「我想把湘宴做到100家店。」

系统展示：

1. 我理解的目标  
2. 建议先解决的三个问题  
3. 将邀请的顾问（不出现 M-XX）  
4. `[开始战略会议]`

禁止：选 Agent / 选模型 / 复杂表单。

---

## 会议三阶段 + 出现感

1. **专家诊断** — 席位先「分析中」，再逐个出现人格化观点  
2. **观点碰撞** — 核心争议 VS  
3. **形成共识** — 决策卡草案 + 接受/继续  

专家文案必须像顾问说话，禁止 `M-PNT: xxxx`。

---

## Agent 隐藏原则

前台只见问题与顾问席。  
M-PNT / M-MKT / M-BIZ / M-ED 仅存于路由与计量。

---

## MVP 边界

只做五页飞轮 + Mission 理解 + 会议三阶段。  
部门看板是企业世界的深入层，不是首页入口。
