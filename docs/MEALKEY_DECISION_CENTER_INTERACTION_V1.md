```text
  → 系统成长
```

## 一、首页：今日经营决策（非 Dashboard）

### 1.1 第一感受（冻结文案心智）

> 今天我的店发生了什么？我最应该关注什么？

### 1.2 首页结构（冻结）

```text
餐启 · MealKey
今日经营决策

{问候}，{老板名}

今日经营状态
  健康度 ★★★★☆  82     ← 无证据则「—」禁止假数
  一句状态：增长质量承压 / …

今天发现
  关键风险  n
  增长机会  n
  执行建议  n

──────── Decision Feed（决策流）────────
  [红] 利润风险 · 人效下降…
  [黄] 增长机会 · 老客复购…
  （3～5 条，主条突出）

[进入今日决策会议]     ← 唯一强 CTA
今日行动 ≤3
```

### 1.3 铁律

- **不要**堆指标；老板缺的是判断。  
- Feed **不是新闻**，是决策流（每条可进入分析）。  
- 健康度无支撑 → 显示「暂无」+ 补全入口。

## 二、Decision Feed（决策流 · 冻结）

每日 **3～5** 条最重要经营事件（Signal / Card 投影）。

### 2.1 条目结构

```text
[风险 | 机会 | 行动]

标题：人效下降正在影响利润

发现：
  过去14天 营业额 +5% · 人工成本 +18%

系统判断：
  增长质量下降

影响：
  预计月利润减少…（无依据则定性，禁止硬编）

可信度：89%

[查看决策分析]
```

### 2.2 规则

| 规则 | 说明 |
|------|------|
| 排序 | 阻断风险 > 高紧急机会 > 行动 |
| 主条 | 第一条视觉加重 = 今日主会题 |
| CTA | 「查看决策分析」→ 开/进 Decision Case |
| 禁止 | 资讯口吻、无 Evidence 的硬影响金额 |

## 三、Decision Room（决策室 · 核心体验）

**不是聊天。** 隐喻：**经营决策会议室。**

### 3.0 顶栏（冻结）

```text
决策问题：是否调整晚餐套餐策略？
影响等级：★★★★☆
决策期限：7天内
决策准备度：68 分（随挑战更新）
```

### 3.1 事实层 —「我们知道什么？」

建立信任：AI 不是猜。

```text
经营事实（每条带来源）
  晚餐销售 ↓12%     来源：POS / 老板脉搏
  客单价   ↓6%      来源：…
  复购率   ↓8%
  竞品数量 ↑3家     来源：外部监测
```

来源枚举：`POS` · `老板输入` · `外部监测` · `历史数据` · `Brain`。

### 3.2 认知层 —「MealKey 怎么看？」

展示理解，**不**直接终局答案。

```text
当前问题不是客流下降，
而是晚餐消费价值感下降。

核心原因：
1. 套餐吸引力不足
2. 竞争价格变化
3. 老客需求变化
```

对应 DIE：Frame + Reason。

### 3.3 七常委 —「决策辩论墙」（冻结）

**禁止**七个头像群聊。

```text
战略官 CSO    支持调整     品牌需保持竞争力
财务官 CFO    反对立即调整 利润空间不足
运营官 COO    条件支持     不能增加复杂度
市场官 CMO    支持         消费者需求正在变化
```

**重点显示冲突**——无冲突则弱化本层或提示「共识较高，仍需路径对照」。

### 3.4 路径模拟层（区别 GPT）

并排/上下 **方案 A / B（/C）**：

```text
若选 A：立即调价
  D30 销量恢复倾向 · D60 成本压力 · D90 利润改善有限
  把握：中（依据：…）     ← 默认分档；高证据才可显示区间%

若选 B：先优化套餐结构
  D30 成本可控 · D90 复购提升
  把握：高（依据：…）
```

**概率纪律：** 与 DIE 一致——无历史匹配禁止大字「成功概率 73%」；有匹配可用区间或分档+依据。

### 3.5 最终裁决层（老板必须参与）

```text
综合判断：推荐先优化套餐结构，暂缓价格调整
决策信心：84 分
主要依据：✓ 数据支持  ✓ 多角色验证  ✓ 风险可控

你的选择：
  [接受建议]  [修改方案]  [坚持原方案]
```

## 四、允许老板反驳（冻结 · 真决策）

点 **坚持原方案** → **风险确认模式**（不可跳过）：

```text
你选择了与系统推荐不同的路径。

历史相似案例提示：（有则显示，无则定性）
  类似情况需额外警惕：组织/现金/…

请补充：为什么选择该方案？（写入 Brain · Founder 决策记忆）

[我了解风险，仍然坚持]  [返回改选]
```

产出：`DecisionChallenge` 确认 + 老板理由 → Brain；Confidence Score 可降但 **尊重裁决**。

## 五、Decision Journey（决策之后 · 冻结）

裁决不是结束：

```text
决策：调整晚餐套餐
状态：执行中

Day 7  完成：新品测试   效果：客单价 +8%（有数才写）
系统学习：该区域更接受套餐化 → Learning → Brain
```

入口：首页弱提示「进行中的决策」或行动页 Journey 卡；详情挂 Case.status = EXECUTING / LEARNING。

## 八、成功标准

1. 首页 10 秒内感知「有人在替我看店」。  
2. Feed 可读完一条完整「发现→判断→影响→可信度」。  
3. 决策室无自由聊开场；五层可滑完。  
4. 辩论墙可见至少一组冲突。  
5. 坚持原方案必须过风险确认并留下理由。  
6. 裁决后能看到 Journey 或今日行动。  

---

**一句话：**  
Decision Center = 每天一场**经营决策会议**——先扫问题，再进会议室把事实、冲突、路径过完，由老板拍板并承诺执行。
