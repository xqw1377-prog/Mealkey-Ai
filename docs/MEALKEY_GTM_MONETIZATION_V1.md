## 〇、战略原则（冻结）

基础设施终局可以服务「所有餐饮人」；**冷启动不能从大而全开始。**

AI 产品规律：

> 先找到一个**高价值 · 高频 · 高痛点**入口，再扩展智能边界。

必须同时具备：

| 构件 | 含义 |
|------|------|
| **切入口** | 第一用户 + 第一场景 |
| **成长路径** | 免费关系 → 能力付费 → 企业系统 |
| **生态扩张** | 老板 → 店长/员工 → 连锁 → 供应链（后置） |

**产品身份（商业话术）：**

| 卖 | 不卖 |
|----|------|
| **你的餐饮经营 AI** / AI 经营顾问（增强判断） | 「AI 聊天」「AI 副总」（替代管理期待） |

**价值一句：**

> 帮餐饮老板做更好的经营判断。

**战略句（与架构对齐）：**

> MealKey = **餐饮行业能力基础设施**（终局）；GTM 阶段对外可先说「餐饮经营 AI」。

## 一、用户分层与切入顺序

```text
创业者 · 老板 · 店长 · 员工 · 连锁企业
```

| 阶段 | 用户 | 为何 |
|------|------|------|
| **P1 切入** | **餐饮老板 / 创业者** | 痛点强 · 决策价值高 · 付费意愿远高于员工课 |
| P2 | 店长 | 门店执行与人效；挂老板账下 |
| P3 | 员工 | Skill 训练；单价低，靠企业包 |
| P4 | 连锁 | 企业经营智能系统（SaaS） |

**P1 不服务「所有角色」当冷启动。**

老板痛点簇（P1）：生意不好 · 人不好管 · 成本上涨 · 不知如何增长。  
决策杠杆：一建议可影响数十万～数百万投资 → 付费区间可支撑「几百～几千/月」级专业能力（具体价目后置标定，不在本文锁死数字）。

## 二、第一产品形态

不是卖 Chat。

卖：**经营判断增强** —— 诊断 · 方案资产 · 目标推进 · 越用越懂你。

服从：交互宪法 · Behavior Engine · Mobile First。

---

## 三、第一批场景（只三刀启动）

| # | 场景 | 用户话 | 输入 | 输出（资产） |
|---|------|--------|------|----------------|
| 1 | **开店决策** | 我要开一家店 | 意图+参数 | 定位·商圈·产品·投资·风险 |
| 2 | **经营诊断** | 为什么赚钱越来越难 | 营业/成本表或口述 | 变量拆解·根因·改善方向 |
| 3 | **菜单优化** | 菜单怎么调 | 菜单/销售 | 结构·毛利·爆品建议 |

三场景足够冷启动；其余能力不得抢 P1 叙事。  
日活飞轮（雷达→决策室→执行→复盘）= **续用与关系**；三场景 = **获客与付费感知**。二者并行：获客用场景，留存用飞轮（见 Core Product Loop / MVP 90 Day）。

## 四、商业模式：基础免费 + 能力付费

类比：OS 建关系，专业能力收费。—— **不是**简单「全站订阅一刀切」为唯一形态。

### 4.1 免费层 · MealKey Core

目的：**建立关系与记忆**，不是白嫖无限专业报告。

含（V1 语义）：

- 自然对话入口（Mobile）  
- 基础经营问答 / 轻诊断框架  
- 用户与门店记忆（合法燃料）  
- 基础分析草稿（可限次或水印深度）

### 4.2 专业能力收费 · Capability Packs

对老板呈现为 **场景能力包 / 专业报告包**，**禁止**呈现为「去商店点财务 Agent」。

| 能力包（示例名） | 对应场景 | 内部可编排能力单元 |
|------------------|----------|--------------------|
| 开店顾问包 | 开店决策 | 定位/投资/选址等（幕后） |
| 经营诊断包 | 经营诊断 | 利润/人效/成本分析（幕后） |
| 菜单优化包 | 菜单 | 产品结构/毛利（幕后） |
| 培训成长包 | Skill（后置） | 陪练与岗位地图（幕后） |

计费形态可组合：包月 · 按项目 · 按深度报告次数（实现层后置；收费 Go 服从 `MEALKEY_COMMERCIAL_DELIVERY_CHECKLIST_V1`）。

### 4.3 企业版（后置）

对象：连锁。  
价值：岗位训练 · 知识/SOP · 员工成长 · 多店经营分析 —— **企业经营智能系统**，SaaS。  
非 P1 冷启动。

## 五、数据飞轮（真正壁垒）

收入是结果；壁垒是**合法沉淀的经营智能数据**。

```text
老板问题 → 经营场景 → 决策过程 → 执行结果 → 经验沉淀
        → Restaurant Intelligence / Memory 更强
```

规模后可知：常见问题 · 有效方案 · 高成功率模式 → 行业 AI 大脑。

**铁律：** 服从 User Intelligence —— `contributeToIndustryModel` 默认关；禁止偷偷跨租户；企业可展示可关闭。

## 六、冷启动策略（冻结）

**不要先做平台。先做「AI 餐饮经营顾问体验」。**

1. 找到约 **100 个**餐饮老板（种子）  
2. **30 天**陪真实问题：菜单 / 利润 / 人员等  
3. 过程沉淀：场景 · 规则 · 模型（进 KG / Skill / Compiler）  
4. 验证命题见下一章 MVP 0.1；工程节奏对齐 MVP 90 Day  

与创始人 30 年经验匹配：**高密度陪跑 > 空泛获客。**

> 先经营者，再岗位与连锁。  
> 卖经营判断，不卖聊天壳，不卖副总幻觉。
