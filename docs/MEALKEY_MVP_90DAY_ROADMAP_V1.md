# MealKey MVP 90天落地路线 V1（冻结）

> **状态：正式冻结（Freeze）— 从设计系统 → 跑通生命体**  
> **日期：** 2026-07-21  
> **上游战略：** `docs/MEALKEY_CORE_PRODUCT_LOOP_V1.md`  
> **唯一场景：** 一家餐厅老板每天打开 MealKey  
> **权威挂载：** `docs/AUTHORITY.md` L0 战略收口  

---

## 〇、90天目标（一个）

跑通并验证第一条完整飞轮：

```text
Identity → RIP → Brain → 日扫/Signal → Radar → Decision Room
        → 裁决 → M-EXEC → 7日复盘 → Brain/Evolution 更准
```

**成功（必须同时满足）：**

1. 种子老板能独立走完「首次建档 → 连续 7 天打开雷达 → 至少 1 次决策签字 → 看到复盘」  
2. 至少 1 条 Signal 带合格证据链，且老板认可「像在看我的店」  
3. 第二天打开率（D2）与「愿意继续用」访谈达标（见 §五）  

**非目标：** 扩 Agent、全渠道自动化运营、多店集团中台、完整 POS 生态。

---

## 一、模块分级：必须真 / 可模拟 / 暂不做

| 模块 | 分级 | 说明 |
|------|------|------|
| Business Identity + RIP 确认门禁 | **必须真实** | 无身份不进驾驶舱；认知卡可确认/修订 |
| Daily Radar 首页（判断书结构） | **必须真实** | 文案与 1+≤3 结构；禁止数字墙 |
| Business Signal Engine（五层+Ranking） | **必须真实** | 包已落地；禁止 UI 内写公式 |
| Signal → 决策室 Brief | **必须真实** | `decisionReadyPath` + 证据摘要 |
| Decision Room + 签字（MKDecision） | **必须真实** | Case.id≡Decision.id；可 LLM，允许启发式降级但须标注 |
| M-EXEC 行动计划创建 | **必须真实** | 签字后**自动**生成行动（补齐断点） |
| 7 日复盘仪式 | **必须真实（产品）** | 第 7 天雷达/Inbox 出现「复盘到了」；规则引擎可简 |
| Restaurant Brain 读写 | **必须真实（薄）** | 身份/决策/复盘回写；UI 不抢首页 |
| 外部情报（点评/小红书/抖音/地图） | **真检索 + 诚实降级** | 公开搜索/配置 SearXNG·SerpAPI；**禁止编造**星级与占比 |
| 点评/抖音官方 API、完整爬虫 | **暂不做** | 合规未就绪则保持检索标签化 |
| POS/客流/人效自动接入 | **V1 可模拟/上传** | Excel/表单周更即可；无数据则 OPERATION 信号降权 |
| 七常委「人格化聊天」 | **不做** | 只做挑战与质量门禁 |
| 更多垂直 Agent / 报表中心 | **不做** | 闸门见核心闭环文档 |

## 二、三阶段节奏（D0 = 冻结日 2026-07-21）

### Phase A · Day 0–30「生命体征」

**目标：** 每天打开有东西看；首次路径不卡死。

| # | 交付 | 真/模 | 验收 |
|---|------|-------|------|
| A1 | Identity → RIP 确认 → `/dashboard` 雷达 | 真 | 未确认不可进驾驶舱 |
| A2 | 日更外采差分 → WorldChange → Signal → Radar | 真 | 有证据则出 primary；无证据诚实空态 |
| A3 | 雷达【进入经营分析】预填 Brief | 真 | 带 topic + 证据摘要 |
| A4 | 决策室可签字落 MKDecision | 真 | 无平行 Case 表 |
| A5 | 种子脚本：1 店演示数据（可关） | 模 | 仅演示/测试；生产默认关 |
| A6 | 外采配置清单（SearXNG/SerpAPI）+ 空态文案 | 真 | 无 Key 不装懂 |

**Phase A 出口：** 内部可完整演示「建档→雷达→进决策室→签字」；录屏 5 分钟。

---

### Phase B · Day 31–60「闭环咬合」

**目标：** 决策后有行动；7 天后有回声。

| # | 交付 | 真/模 | 验收 |
|---|------|-------|------|
| B1 | 签字后自动创建 M-EXEC ActionPlan | 真 | 无需手动翻到「行动」才生成 |
| B2 | 雷达/Inbox：「今天一件事」= 执行或决策 | 真 | 与 primary 或未完成行动对齐 |
| B3 | **D+7 复盘卡片**（决策日起算） | 真 | 到期出现；点进复盘三问 |
| B4 | 复盘结果写回 Brain + Evolution 信号 | 真 | 可在「我的餐厅」/成长侧看到一条学习 |
| B5 | Signal Promote Gate 与手动升格统一 | 真 | 证据不足不可静默开会 |
| B6 | 经营数据：周上传模板（客流/营业额/客单） | 模→真 | 有上传则 OPERATION 信号可信度↑ |

**复盘三问（冻结）：**

1. 当时判断对不对？  
2. 执行做了没有？  
3. 生意有没有按预期变化？  

**Phase B 出口：** 同一决策从签字到 D+7 复盘可追踪；种子用户完成 ≥1 次全闭环。

---

### Phase C · Day 61–90「种子验证」

**目标：** 外人愿意每天打开；收敛真需求。

| # | 交付 | 真/模 | 验收 |
|---|------|-------|------|
| C1 | 种子用户 5–10 家单店（同城或同品类优先） | — | 签署数据使用说明 |
| C2 | 每人连续使用 ≥14 天；至少 2 次「进入分析」 | — | 埋点或手记 |
| C3 | 访谈：是否感觉「在看我的店」 | — | ≥60% 明确肯定 |
| C4 | D2 / D7 打开率记录 | — | 定基线（不设虚高 KPI） |
| C5 | 缺口清单只许三类：证据可信 / 行动摩擦 / 复盘价值 | — | 禁止据此扩 Agent |
| C6 | 90 天结论纪要：Go / Pivot 证据层 / Kill 范围 | — | 写入 AUTHORITY 附录或本文件修订说明 |

**Phase C 出口：** 书面结论——飞轮是否成立；下一季度只加深证据与摩擦，不开新层。

## 三、工程主线（唯一）

```text
修断点 > 真数据 > 复盘仪式 > 种子验证
```

**优先修的已知断点（相对现状）：**

1. ~~Council 结案 → M-EXEC **自动**衔接~~ **已落地**（`confirmFromMeeting` / `founderDecide` 自动 `createExecutionFromDecision`）  
2. ~~7 日复盘 **产品入口**~~ **已落地**（`d7-review.ts` → DailyScan.reviewDue → 雷达首屏）  
3. ~~外采空态与配置；禁止假百分比~~ **已落地**（`radar.emptyIntelNote` 诚实空态）  
4. ~~Brain/Identity 对 Radar 相关性注入~~ **已落地**（`extractRestaurantContextForSignals` → Signal Engine）  
5. ~~雷达证据进 Brief~~ **已落地**（`evidenceSummary` → DecisionRoom）  

**明确不做的工程：** 新 Prisma 大表、新顾问席、首页改造成多入口门户。

## 四、数据策略（种子期）

| 数据 | 策略 |
|------|------|
| 品牌/门店/城市/客单带 | Identity 必填 |
| 点评/小红书/抖音提及 | 公开检索 + 来源标签；无结果显示「未观察到公开变化」 |
| 竞品/地图 | 检索 + 老板确认；不编造「新开 N 家」精确数除非有 sourceRef |
| 营业额/客流 | 周 CSV/表单；缺失则弱化 OPERATION，强化 CUSTOMER/COMPETITION |
| 决策与复盘 | 100% 真实落库（既有 Decision / outcome） |

## 五、种子用户如何验证

**招募画像：** 单店或 ≤3 店老板；本人天天看点评；愿意每周给 1 次反馈。

**验证动作（最小）：**

| 周次 | 动作 |
|------|------|
| W0 | 建档 + 确认认知卡 + 讲解「雷达不是报表」 |
| W1 | 只要求每天打开看判断句；收集截图/口述 |
| W2 | 引导 1 次从雷达进决策室并签字 |
| W3–W4 | 跟 1 次 D+7 复盘；访谈三问 |

**合格信号：** 「不用我问它就知道店里出事了」类原话；或主动分享雷达截图给合伙人对齐。

**淘汰信号：** 只当 ChatGPT 用；或只要报表数字墙。

## 六、周节奏（建议）

- **周一：** 看种子漏斗与空态率（无证据日占比）  
- **周三：** 只修闭环断点（A/B 表内）  
- **周五：** 1 次老板访谈或内部走查录屏  
- **禁止：** 周五开新能力设计评审
