# MealKey Skill Engine V1.0（餐饮能力引擎 · 冻结）

> **状态：正式冻结（Freeze）— 知识 → 能力；使 MealKey 从「餐饮知识 AI」升级为「餐饮行业能力成长系统」**  
> **日期：** 2026-07-23  
> **权威挂载：** `docs/AUTHORITY.md` L0 认知护城河  
> **上游：** Intelligence Model · Knowledge Graph（Role→Outcome 因果边）· Persona  
> **协同：** User Intelligence（经营者偏好/决策燃料，≠岗位陪练）· Brain（门店/品牌实例）· Business / Goal Compiler（经营结果连接）  
> **明确不做：** 传统课程库 SaaS；考试分数冒充能力；第五顾问席 / 第七 Runtime；Agent Protocol「Decision Skill」包混淆为本引擎；P1 未验证前全岗位培训平台；普通聊天当能力燃料。

## 〇、生命线（冻结）

> **理解 ≠ 改变。知道 ≠ 会做。**

| 行业现状 | MealKey 立场 |
|----------|--------------|
| 菜品知识 · 服务流程 · 制度 · 经验分享很多 | 缺的是 **真实场景下做出正确动作的能力** |
| GPT：我知道餐饮知识 | MealKey：**我知道岗位应如何工作，并帮你成为更好的餐饮人** |
| 回答问题 | **培养能力** |

**一句话定义：**

> Skill Engine = 把岗位知识编译为「场景中的可练习行为」，并连接经营结果的能力成长引擎。

**能力形成链（冻结）：**

```text
知识 → 理解 → 场景 → 行为 → 反馈 → 能力
```

**战略身份：**

> MealKey ≠ 餐饮 SaaS ≠ 餐饮 ChatGPT。  
> 做到本层 = **餐饮行业的 AI 能力基础设施**（老板 / 店长 / 员工分阶段接入）。

---

## 一、核心公式（冻结）

```text
Skill = Role × Scenario × Behavior × Outcome
```

能力不是孤立词条；四维缺一不可。

| 维 | 问什么 | 示例（服务员 · 推荐） |
|----|--------|------------------------|
| **Role** | 谁需要？ | 服务员 |
| **Scenario** | 何时用？ | 点单 / 推荐 |
| **Behavior** | 具体怎么做？ | 观察人数→判断场景→问偏好→推荐组合→确认反馈 |
| **Outcome** | 影响什么经营结果？ | 推荐率→加菜率→客单→营业额 |

**反例（禁止当 Skill）：** 「要主动推荐菜品」——只有口号，无 Scenario/Behavior/Outcome。

---

## 二、与图谱 / 经营引擎的连接（冻结）

Skill **必须**挂 Knowledge Graph 的 Role→Outcome 边，最终连 Business Engine：

| 岗位 | 能力结果链（示例） |
|------|-------------------|
| 服务员 | 推荐能力 → 推荐率 → 客单 → 营业额 |
| 厨师 | 出品稳定 → 满意度 → 复购 |
| 店长 | 排班/管理 → 人效 → 利润 |
| 老板 | 决策质量 → 资源分配 → 利润/风险（连 DIE，非替拍板） |

**铁律：** 员工能力不可独立于经营；无 Outcome 挂点的「软技能课」不得进 V1 主路径。

## 三、岗位能力地图（Skill Map）

每个岗位一份地图；能力可再拆行为链。

### 3.1 服务员 Skill Map（V1 样板 · P2 竖切）

```text
服务员
├ 基础能力
├ 产品能力
├ 沟通能力
├ 销售能力          ← 优先竖切
├ 服务体验能力
└ 问题处理能力
```

**销售能力拆解：**

```text
识别客户需求 → 推荐产品 → 搭配销售 → 处理异议 → 促成购买
```

### 3.2 店长 / 老板（P1 优先语境）

| 岗位 | P1 优先能力族 |
|------|----------------|
| 店长 | 排班与人效 · 客诉升级 · 今日经营解读（连雷达/Compiler） |
| 老板 | 定位判断 · 利润诊断追问 · 开店/菜单决策路径（连 Decision Pattern） |

P1 **先验证老板/店长能力陪练与经营连接**；全量服务员/厨师地图后置，但契约与公式现在冻结。

## 四、能力等级（非考试分）

| 等级 | 名称 | 含义 | 示例（接待） |
|------|------|------|----------------|
| L1 | 知道 | 知道规则/菜单/标准 | 能说出主推菜 |
| L2 | 会做 | 能完成基本任务 | 正常接待顾客 |
| L3 | 优秀 | 能主动创造结果 | 提升客单 |
| L4 | 教练 | 可培养别人 | 带新人推荐话术 |
| L5 | 专家 | 能优化系统 | 改进门店推荐流程 |

升级依据：**场景练习表现 + 工作反馈 +（可得时）经营代理指标**；禁止单一笔试定级。

## 五、AI 训练场（实时陪练 · 非课程库）

区别于传统培训「给课件」：

```text
打开 MealKey → AI 发起场景练习 → 员工应答 → AI 按标准评价 → 写入反馈
```

**评价维度（V1 最小）：**

1. 推荐/动作是否合理（合 Scenario + 门店/品牌实例）  
2. 话术是否自然、可执行  
3. 是否符合品牌定位（Brain / Brand 节点）

**样例剧本：** 「今天练习家庭聚餐推荐」→ 顾客：「两个人，有老人」→ 员工答 → 评分与改进点。

实现形态：对话回合内 Coach Mode；**不是**独立 LMS 站点抢入口（P1 挂 Mobile Agent / 店长工作面）。

## 六、能力反馈闭环（冻结）

```text
学习 → 练习 → 工作 → 反馈 → 优化 → 升级
```

示例：过去 30 天画像  
- 菜品知识强 · 推荐弱 · 客诉弱 → 生成 **个人提升计划**（优先 Scenario 清单）

**燃料边界（与 User Intelligence 分轨）：**

| 引擎 | 合法燃料 |
|------|----------|
| Skill Engine | 陪练回合结果 · 岗位任务完成/纠偏 ·（可选）门店经营代理指标 |
| User Intelligence | 决策选择 · 修改/推翻 AI · M-EXEC 完成率 · 预测 vs 实际 |

禁止：把闲聊当岗位能力升级依据；禁止未授权把员工陪练写入 Industry / 跨租户。
