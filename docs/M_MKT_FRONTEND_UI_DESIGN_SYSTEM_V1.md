# M-MKT Frontend UI Design System V1

> 模块：M-MKT
> 产品名：市场机会中心
> 英文名：Market Opportunity Center
> 文档角色：视觉与交互规范稿

## 一、整体视觉定位

这一层非常关键。

因为前面解决的是：

**功能结构**

现在解决的是：

**用户感知**

我们的目标不是做一个高级数据后台。

而是做一个：

> AI 市场战略咨询会议室。

冻结：

## M-MKT = Strategy Room

四个 Agent 对应四种空间：

| Agent | 空间隐喻 | 用户感知 |
| --- | --- | --- |
| M-MKT | 市场战略会议室 | 判断机会 |
| M-PNT | 品牌战略会议室 | 找定位 |
| M-BIZ | 商业模型会议室 | 算生意 |
| M-ED | 股权治理会议室 | 定规则 |

所以 M-MKT 不应该像：

- 数据驾驶舱
- BI 报表
- Excel 分析

应该像：

- 战略顾问桌
- 投资委员会
- 麦肯锡项目室

## 二、视觉关键词

确定关键词：

```text
专业
克制
判断
洞察
可信
```

避免：

- 科技炫酷
- 大量渐变
- AI 机器人感
- 聊天窗口感

## 三、页面整体布局

桌面端：

```text
------------------------------------------------

Header

市场机会中心

长沙 × 湘菜


------------------------------------------------


       Journey Timeline


①理解
②扫描
③发现
④讨论
⑤决策


------------------------------------------------


              主工作区


------------------------------------------------


              决策区


------------------------------------------------
```

### 布局原则

- `Header` 提供项目和当前阶段
- `Journey Timeline` 提供咨询路径感
- 主工作区只展示当前阶段任务
- 决策区负责结果收口和下一步路由

## 四、颜色体系

M-MKT 采用：

## 深蓝 + 米白 + 金色强调

原因：

市场判断不等于创意设计。

它需要：

**信任**

### Primary

深战略蓝：

```text
#172554
```

代表：

战略

### Background

温暖白：

```text
#FAFAF8
```

### Accent

机会金：

```text
#C9A227
```

用于：

- 机会评分
- 关键发现
- 推荐动作

### Risk

风险橙：

```text
#EA580C
```

### Safe

进入绿：

```text
#16A34A
```

### 色彩使用规则

- 蓝色用于结构、标题、阶段、咨询容器
- 金色只用于“当前机会”和“重要高亮”
- 绿色只用于明确建议进入
- 橙色只用于风险和争议提醒

## 五、Header 设计

不是普通标题。

设计结构：

```text
------------------------------------------------


← 返回项目


市场机会中心


Market Opportunity Center


长沙湘菜项目


阶段：

市场判断


------------------------------------------------
```

右侧状态：

```text
AI 战略顾问

● 在线分析
```

组件：

```text
MarketHeader
```

### Header 原则

- 不讨论模型和 Agent 名词
- 只表达“谁在判断什么”
- 用阶段状态增强咨询过程感

## 六、Journey Timeline 设计

这是用户理解流程的核心。

不是普通步骤条。

设计形态：

```text
     ●
     |
     |
     ●
     |
     |
     ●


市场理解

市场扫描

机会判断

专家讨论

进入决策
```

### 状态规则

- 当前节点：金色高亮
- 已完成：深蓝
- 未开始：灰色

### 移动端

变为横向：

```text
理解 → 扫描 → 判断 → 决策
```

### 组件

```text
MarketJourneyTimeline
MKTimeline
```

## 七、机会 Hero 设计

首页核心不是仪表盘。

不要大圆环。

采用：

## 战略判断卡

结构：

```text
--------------------------------

市场机会判断


82

存在结构性机会


--------------------------------


一句话：

成熟品类正在发生消费迁移


--------------------------------


最大风险：

传统竞争者密度过高


--------------------------------
```

原则：

- 数字不是重点
- 判断才是重点

### 组件

```text
MarketOpportunityHero
MKDecisionCard
```

## 八、评分设计

六维评分不要直接展示复杂雷达。

V1 展示：

```text
机会形成因素


需求机会       85
竞争空间       70
消费变化       90
进入窗口       80
经营成立       75
```

这更像咨询报告中的：

> 关键判断因素

### 设计要求

- 评分列表优先于图表
- 每个分数必须配一句判断
- 不要为了“高级感”牺牲可解释性

## 九、市场信号卡设计

这是 AI 智能感来源。

卡片结构：

```text
--------------------------------

发现一个市场信号


年轻消费者正在重新定义湘菜场景


为什么：

过去：

家庭聚餐


现在：

朋友社交


影响：

新的品牌机会


可信度 88%

--------------------------------
```

重点不是数据。

而是：

**洞察**

### 组件

```text
MarketInsightCard
MKInsightCard
```

## 十、专家委员会 UI

这里要重点设计。

不要头像聊天。

设计成会议桌。

桌面布局：

```text
                 市场战略专家


                      ▲


经营专家  ◀      项目      ▶ 投资专家
```

每个专家是一张判断卡。

### 战略专家

```text
关注：

趋势变化


判断：

机会存在
```

### 经营专家

```text
关注：

落地难度


判断：

竞争风险高
```

### 投资专家

```text
关注：

规模化


判断：

需要验证复制性
```

### 组件

```text
MarketCouncilRoom
MarketExpertCard
```

## 十一、专家争议动画

这是高感知部分。

流程必须是：

```text
专家观点出现

↓

冲突线连接

↓

AI 总结

↓

形成共识
```

例如：

```text
观点冲突


战略专家

进入窗口打开


        VS


经营专家

成本压力过高



AI 综合判断：

机会存在，
但必须改变进入方式。
```

这会让用户感觉：

> AI 在思考

### 组件

```text
MarketDebatePanel
MarketConsensusCard
```

进入品牌定位中心


================================

### 组件

```text
MarketDecisionCard
MarketNextAction
```

## 十三、交互细节

### 1. AI 分析过程

不要只显示 loading。

改成：

```text
正在完成：

✓ 消费趋势分析

✓ 竞争结构分析

● 案例匹配中

○ 专家讨论
```

### 2. 用户确认节点

每个关键判断都增加：

```text
这个判断符合你的观察吗？

[符合]

[需要调整]

[补充信息]
```

目的：

- 训练用户
- 同时训练 AI

### 3. Stream 视觉

- 进度不要闪烁
- 新 insight 卡片要渐进出现
- 专家观点按顺序进场

## 十四、移动端设计

手机不是缩小版。

要重新排序。

顺序：

```text
机会判断

↓

最大风险

↓

关键发现

↓

专家观点

↓

进入建议
```

首页第一屏必须看到：

```text
82 分

存在机会

最大风险 xxx
```

### 移动端原则

- 不使用横向表格
- 不并排放置信息密卡
- 结论先于解释
- CTA 始终在可触达区域

```text

↓

市场顾问进入

↓

品牌顾问进入

↓

商业顾问进入

↓

治理顾问进入
```
