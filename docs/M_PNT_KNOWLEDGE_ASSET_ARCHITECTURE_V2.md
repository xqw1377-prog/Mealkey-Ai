# M-PNT Knowledge Asset Architecture V2

# 品牌定位智能资产体系设计

## 一、核心目标

前面的 Runtime 和 Workshop 解决了：

> AI 如何组织一次品牌定位会议。

但还没有解决：

> 为什么 AI 能像一个真正的品牌战略专家一样判断？

真正的壁垒不在 Agent 数量。

而在：

**知识资产 + 判断模型 + 案例经验 + 用户反馈闭环。**

所以 M-PNT 的核心不是：

- 三个专家 Prompt

而是：

> 三个专家背后的“认知系统”。

---

# 二、M-PNT 知识资产总架构

整体分六层：

```text
                M-PNT Intelligence Layer

                         ↓

              Brand Positioning Brain


                         ↓


┌────────────────────────────┐
│ 1. 定位理论知识库             │
│ 2. 餐饮案例知识库             │
│ 3. 品类竞争知识库             │
│ 4. 消费者认知知识库           │
│ 5. 失败案例知识库             │
│ 6. 决策反馈学习库             │
└────────────────────────────┘


                         ↓

              Expert Reasoning Engine


                         ↓

              Positioning Decision
```

这六层不是并列资料堆。

它们共同组成：

- M-PNT 的判断底座
- 三专家的差异化知识入口
- 决策反馈的长期学习闭环

---

# 三、第一层：定位理论知识库

## 目标

让 AI 不是“模仿专家”。

而是：

拥有不同理论体系的判断框架。

---

## 3.1 Ries 定位体系

知识结构：

```text
Ries Framework

├ 品类定义
├ 第一认知
├ 心智入口
├ 领导者战略
├ 新品类创造
├ 竞争替代
```

---

例如：

输入：

> 做一个新的湘菜品牌

Ries Agent 不直接回答：

“做年轻化湘菜”。

而先判断：

```text
问题：

湘菜这个品类是否已经被占据？

如果已经成熟：

不要进入旧认知。

寻找：

新的消费类别。
```

---

## 存储形式

```json
{
  "expert": "Ries",
  "principle": "领先品牌必须占据第一个认知位置",
  "evaluation": [
    "是否创造新品类",
    "是否拥有第一概念",
    "是否容易记忆"
  ]
}
```

---

# 四、第二层：Trout 竞争定位库

核心是回答：

> 你在消费者心智中打败谁？

---

结构：

```text
Competition Positioning

├ 竞争对象
├ 差异空间
├ 心智空位
├ 替代关系
├ 攻击策略
```

---

案例：

海底捞：

不是：

“火锅很好吃”。

而是：

服务体验第一。

---

蜜雪冰城：

不是：

“便宜奶茶”。

而是：

高频低价快乐。

---

AI 判断必须找到：

```text
你的品牌：

不是比别人更好。

而是在某个维度成为不同。
```

---

# 五、第三层：叶茂中消费者洞察库

这是餐饮最重要的一层。

因为餐饮购买不是纯理性。

---

结构：

```text
Consumer Insight

├ 表层需求
├ 隐性需求
├ 情绪驱动
├ 消费场景
├ 身份表达
```

---

例如：

家庭宴请。

表面上是：

吃饭。

深层其实是：

```text
我要证明：

我是一个照顾家庭的人。
```

---

AI 输出不应只是：

“家庭用户”。

而应形成：

```text
消费者购买的不是湘菜。

购买的是：

一次让家人满意的关系表达。
```

---

# 六、第四层：餐饮品牌案例库

这是 M-PNT 最大的资产之一。

不能只是：

品牌资料。

必须是结构化战略案例。

---

## Brand Case Schema

```typescript
BrandCase {
  brandName
  category
  city
  stage

  positioning
  targetCustomer
  competitiveEnemy

  breakthroughPoint

  execution

  result

  failureReason
}
```

---

案例：

## 喜茶

不是保存：

“喜茶介绍”。

而是保存成战略知识。

例如：

```text
问题：

传统奶茶低端化。

机会：

创造新茶饮品类。

定位：

年轻人的高品质茶饮。
```

---

# 七、第五层：餐饮品类认知库

因为定位不能脱离品类。

需要建立：

## Category Brain

例如：

湘菜：

```text
品类：

湘菜


消费者认知：

重口味
家庭消费
地域属性


竞争问题：

全国品牌弱


机会：

？
```

---

火锅：

```text
成熟品类

竞争高度集中

需要：

场景创新
体验创新
价格创新
```

---

# 八、第六层：失败案例库

这个非常重要。

高手不是只知道成功案例。

而是知道：

为什么失败。

---

结构：

```typescript
FailureCase {
  brand
  strategy
  why_failed
  wrong_assumption
  lesson
}
```

---

例如：

某品牌：

定位：

年轻高端湘菜。

失败原因：

```text
消费者认为：

湘菜不需要高端化。
```

---

AI 在未来应该能够主动提醒：

> 这个方向过去已有类似失败案例。

---

# 九、知识如何进入三个专家

不是共享同一坨知识。

而是三位专家读取不同视角。

---

## Ries Agent

重点读取：

```text
定位理论

品类案例

新品类案例
```

---

## Trout Agent

重点读取：

```text
竞争案例

定位战争

品牌对抗案例
```

---

## 叶茂中 Agent

重点读取：

```text
消费者洞察

传播案例

情绪消费案例
```

---

所以：

三个 Agent 不一样，不是因为 Prompt 文案不同。

而是因为它们背后的知识入口、判断角度、案例读取路径不同。

---

# 十、知识调用机制

不是一次全部塞给 LLM。

而是采用：

## Retrieval + Reasoning

流程：

```text
用户问题

↓

判断问题类型

↓

调用知识模块

↓

生成专家判断

↓

形成观点
```

---

例如：

用户：

“我想做年轻湘菜。”

系统识别：

```text
category:
湘菜

question:
positioning
```

然后调用：

```text
湘菜品类库

年轻消费洞察库

竞争案例库

Ries 知识
Trout 知识
叶茂中知识
```

---

# 十一、用户反馈学习层

这是未来真正的壁垒。

每次定位结束后，系统都记录：

```text
用户选择

为什么选择

后来结果

实际经营数据
```

---

形成：

## Positioning Learning Loop

```text
定位建议

↓

用户执行

↓

经营结果

↓

反馈

↓

模型优化
```

这意味着 M-PNT 的知识资产不是静态知识库，而是持续增长的判断系统。

---

# 十二、M-PNT 最终能力形成

未来，老板进入系统说：

“我想做一家社区湘菜。”

AI 不会只是继续追问：

“请告诉我更多。”

而是已经知道：

```text
社区湘菜：

历史案例：

成功模式：

失败原因：

竞争环境：

消费者心理：

可能定位方向：
```

然后主动引导：

> 根据你的城市、资源和目标，我认为你有三个方向，但第二个方向风险最高。
