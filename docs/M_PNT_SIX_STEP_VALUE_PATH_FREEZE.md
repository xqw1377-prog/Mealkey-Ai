# M-PNT 六步价值路径冻结（纠偏）

> 冻结日：2026-07-15  
> 状态：**权威产品路径**（覆盖「填表门禁 → 丢文档」的错误主路径）  
> 原则：**过程可感价值 > 表单完整度；交付是决策结果，不是唯一价值点。**  
> 权威入口：见 `docs/AUTHORITY.md`

---

## 0. 问题定性

当前实现的错误逻辑：

```text
长时间收集信息 → 门禁/术语/多按钮 → 最后生成一份文档
```

用户感知：

- 被折腾、被考试
- 中间没有「咨询机构在干活」的感觉
- 终点只有文档 → **无价值感**

正确逻辑（本文件冻结）：

```text
① 采集信息（极少、精准）
  → ② 市场调研（机构干活，用户看洞察）
  → ③ 三位顾问各出定位策略（里斯 / 特劳特 / 叶茂中）
  → ④ 四方会议室（三顾问 + 用户）讨论博弈达成共识
  → ⑤ 确认定位策略 → 生成《定位策略报告》用户签字确认
  → ⑥ 生成定位执行路径（时间节点 + 落地执行方案）
```

---

## 1. 六步主路径（用户可见，不可改名乱序）

| 步 | 代号 | 用户感知 | 系统在干什么 | 本步产出（必须可感） |
|----|------|----------|--------------|----------------------|
| 1 | `INTAKE` | 「它懂我店」 | 精简采集（≤5 高信号题） | 项目简报（内部）；用户侧只看到「已了解」摘要卡 |
| 2 | `MARKET_RESEARCH` | 「机构在调研」 | 品类/客人/竞争调研合成 | **市场调研报告卡**（趋势、战场、空位、风险） |
| 3 | `THREE_ADVISORS` | 「三位大师在出策」 | 三理论 Agent 各出一策 | **三份定位策略卡**（有冲突、可对比） |
| 4 | `WAR_ROOM` | 「我在会议室拍板」 | 四方会议：挑战→取舍→共识 | **会议纪要 + 共识草案**（用户参与投票/拍板） |
| 5 | `STRATEGY_LOCK` | 「方案定了」 | 冻结定位合同 + 策略报告 | **《定位策略报告》** + 用户确认 |
| 6 | `EXECUTION_PATH` | 「知道怎么干」 | 时间轴落地 | **90 天执行路径**（节点、责任、验收） |

**禁止**：把步骤 2–4 藏成后台分析，只在步骤 5 甩文档。  
**禁止**：用「完成门禁清单」替代「看见顾问在开会」。

## 2. 与旧阶段的映射（内核资产可保留，主 UX 必须换皮）

旧 `BrandProjectStage`（P0 门禁骨架）**降级为内部子状态/资产管道**，不再作为老板主叙事：

| 六步 | 可复用旧资产/能力 |
|------|-------------------|
| 1 INTAKE | Discovery + BrandBrief（五问）+ 事实种入 |
| 2 MARKET_RESEARCH | Category / Consumer / Competitive 分析合成 → `MarketResearchPack` |

## 3. 三位顾问（冻结人格，不是三个文案机）

| 顾问 | 模型 | 必须回答的一句话 |
|------|------|------------------|
| 里斯 | 心智占位 | 用户为什么记住你？ |
| 特劳特 | 竞争空位 | 为什么选你不选别人？ |
| 叶茂中 | 情绪/场景 | 为什么愿意喜欢/传播你？ |

每步 3 必须各自输出完整「定位策略卡」：一句话定位、给谁、战场、差异、风险、不做什么。  
允许且鼓励 **互相打架**；伪共识视为失败。

---

## 4. 四方会议规则（步骤 4）

参与方：**里斯 · 特劳特 · 叶茂中 · 用户（老板）**。

流程冻结：

1. 各顾问陈述本策（已在步骤 3 生成）
2. Cross-Fire：互相挑战（引用对方策略缺口）
3. 用户表态：倾向 A / B / C / 折中
4. 主持人（系统）提出取舍草案
5. 用户确认共识 → 才能进步骤 5

没有用户拍板，**不得**自动锁定位。

---

## 5. 交付物层级（价值感知顺序）

用户必须先感知过程资产，再看到终稿：

1. 市场调研摘要（步 2）
2. 三策对比（步 3）
3. 会议共识（步 4）
4. 《定位策略报告》（步 5）
5. 执行路径时间轴（步 6）

终稿仍对标 Interbrand/Landor 签字级；但 **终稿不是唯一交付**。

---

## 6. 交互铁律（与「别折腾老板」一致）

1. 步 1 只问不可推断信号；其余系统推断。
2. 步 2–3 默认自动跑完，用户以「阅读 + 确认」为主。
3. 步 4 才是用户主战场（点选/表态/拍板），不是填空作文。
4. 黑条「下一步」始终对应六步之一，不暴露内部门禁术语。
5. 旧表单（六段 For/Who、证据审阅等）全部收入「高级/顾问后台」，默认折叠。
