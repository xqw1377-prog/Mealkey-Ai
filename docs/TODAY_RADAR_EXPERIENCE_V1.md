# TODAY_RADAR_EXPERIENCE_V1（冻结）

> **状态：正式冻结（Freeze）— 日活入口体验**  
> **日期：** 2026-07-20  
> **产品一句：** **今日经营雷达：每天替老板观察生意变化，发现风险和机会，并告诉老板最值得关注的一件事。**  
> **配套：** `BUSINESS_SIGNAL_ENGINE_V1` · `EVIDENCE_CHAIN_PROTOCOL_V1`

---

## 〇、它是 / 不是

| ✅ 是 | ❌ 不是 |
|------|--------|
| AI 经营观察员 | 数据中心 |
| 每天为什么打开 MealKey | 经营分析正文本身（那是下一层） |
| 变化 → 影响 → 建议入口 | 行业资讯流 |
| 进入经营分析的入口 | 七常委 / 决策室本身 |

七常委是深度能力；**雷达是入口与习惯。**  
经营分析标准页见：`docs/BUSINESS_ANALYSIS_OUTPUT_TEMPLATE_V1.md`（雷达 ≠ 分析 ≠ 决策室）。

## 一、首页信息架构（冻结）

```text
餐启 · 今日经营雷达
  问候
  「我昨天观察你的生意，发现 N 个变化。」

  ── 今天最值得关注（仅 1 条）──
  标题 + 重要程度星级
  为什么发现（证据链摘要）
  我的判断
  [进入分析]

  ── 其他变化（≤4）──
  🟡 竞争 / 🟢 机会 / … 一行标题

  ── 经营状态（四维趋势）──
  用户感知 · 市场位置 · 经营效率 · 增长机会
```

### 首屏原则

1. **不要数字墙**；顶部是判断句，不是 KPI  
2. **一件事原则**：最值得关注永远只有一条  
3. 每条可点，必须先进 **经营分析**（再决定是否升格决策室）  
4. 刷新雷达 = 强制日更观察，不 invent 假变化

## 二、老板语言（冻结）

| 禁止 | 必须 |
|------|------|
| 「数据异常 / 指标波动」 | 「你的服务体验正在影响复购」 |
| 「建议关注相关模块」 | 「今天检查晚市 18–20 点出餐」 |
| 无来源的断言 | 「为什么发现」+ 证据链 |

---

## 三、交互

| 动作 | 结果 |
|------|------|
| 进入经营分析 / 查看变化 | `businessAnalysisPath` + `BusinessAnalysisPacket` |
| 分析页「升格到决策室」 | `decisionReadyPath` + VoiceBrief |
| 刷新雷达 | RIP / Observer 日更 |
| 自己发起决策 | 次级入口，不抢首屏 |

---

## 四、与经营分析 / Decision Room

```text
雷达发现 → 经营分析（标准模板）→（可选）决策室 → 七常委
```

雷达**不**替代分析页，也**不**替代七常委；只保证「每天有理由打开，并知道下一步点哪里」。  
禁止在决策室用「今日已采齐」冒充采集完成。
